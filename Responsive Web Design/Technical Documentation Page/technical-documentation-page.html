Introduction

Java is an Object-Oriented programming language that has gained much popularity due to its high level of flexibility, as well as its beginner-friendly nature. The four main concepts of object-oriented programming introduced by Java are:

  • Abstraction
  • Polymorphism
  • Inheritance
  • Encapsulation

Or APIE, for short.

The purpose of this documentation is to give a basic tutorial on how to use Java and explain a few of its concepts.

Hello World

Here's an example of a simple Hello World program written in Java:

Application.java

public class Application {
public static void main(String args[]) {
System.out.println("Hello World!");
}
}

Output

Hello World!

Classes & Objects

Classes

A class in Java is essentially a container for all the data and functions that relate to a specific concept or component. Typically, class names are meant to be informative, letting people know what the data pertains to, in this case, the Application.

Access Modifiers

The keyword that comes before "class" is known as an access modifier. Access modifiers tell the program which other parts of the application are able to see and potentially modify their data. In this case, our "public" access modifier is telling the program that this class can be seen from anywhere else in the application. Here are some examples of other types of access modifiers:

  • private
  • protected
  • default

In general, it is best practice to label classes and methods as "public" so that they can be seen and used, and variables as "private" so that they cannot be modified directly.

Objects

An object is an instance of any given class. If you think of a class like a blueprint, like a blueprint for a house, an object of that class would be the house. It is the physical implementation that contains the data outlined in its class.

We wouldn't create an object of the class Application, because it is our main class where the program starts. However, if we added another class, we could decide to use it in our main class by creating an object. For example:

Dog.java

public class Dog { }

Application.java

public class Application {
public static void main(String args[]) {
System.out.println("Hello World!");
// Here we are creating a new object of the Dog class called "dog"
Dog dog = new Dog();
System.out.println("A new dog was born!");
}
}

Output

Hello World!
A new dog was born!

Methods

A method in Java is a set of steps for the program to follow in order to perform an action. In other words, it's Java's version of a function. Recall that in Application.java, we have a method called "main."

In this method, we print out a message, instantiate a Dog object, and print out another message to confirm this. But what if the dog needs to get our attention? We can add a method in the Dog class to accomplish this.

Dog.java

public class Dog {
public void bark() {
System.out.println("Woof!");
}
}

In order to use this method, we must call it in our main method via the dog object:

Application.java

public class Application {
public static void main(String args[]) {
System.out.println("Hello World!");
// Here we are creating a new object of the Dog class called "dog"
Dog dog = new Dog();
System.out.println("A new dog was born!");
// And here we are calling its method, "bark"
dog.bark();
}
}

Now when we run the program, we can see (or perhaps "hear") that the dog is barking!

Output

Hello World!
A new dog was born!
Woof!

Variables

In Java, like in other programming languages, a variable is a user-defined container for storing data. They are typically helpful when you want to keep track of certain information, like a person's name or how many miles a car has traveled.

Every variable must have a data type in order for the program to know what kind of data will be stored. Some examples of data types include:

  • int
  • double
  • boolean
  • String
  • Class
  • integer
  • decimal
  • true/false
  • text
  • object

We've actually already created a variable in our Application class:

Dog dog = new Dog();

Our variable "dog" has a data type of Class, or more specifically, Dog. That's why we were able to assign it to an object of the Dog class. But if we were to try:

int dog = new Dog();

The code would not compile, because the program was expecting data of type "int" or "integer", and thus could not store data of type "Dog".

Now, let's take a look at our Dog class:

Dog.java

public class Dog {
public void bark() {
System.out.println("Woof!");
}
}

They can bark now, but there's not much else to them. We don't know their name, their age, or even what their favorite toy is. We can fix that by adding some variables:

Dog.java

public class Dog {
private String name = "Molly";
private int age = 3;
private String favoriteToy = "tennis ball";

public void bark() {
System.out.println("Woof!");
}
}

Getters & Setters

Great! Our dog's name is Molly, she's three years old, and her favorite toy is a tennis ball. But notice how these variables were made "private". This means we cannot directly access them from our main method, like we could with "bark".

// This would not work.
System.out.println(dog.name);

Instead we must create a set of helper methods in the Dog class to get this information for us. We call these methods "getters".

Dog.java

...
public int getName() {
return name;
}
public int getAge() {
return age;
}
public String getFavoriteToy() {
return favoriteToy;
}

And now we can call the getters from our main method

Application.java

public class Application {
public static void main(String args[]) {
...
// Print the dog's information using getters
System.out.println(dog.getName() + " is " + dog.getAge() + " years old, and their favorite toy is a " + dog.getFavoriteToy() + "!");
}
}

Output

...
Molly is 3 years old, and their favorite toy is a tennis ball!

Now let's imagine that Molly tore up her favorite tennis ball, and now she has a new favorite toy. Like before, we can't change them directly.

// The variables are still private, so this wouldn't work either.
dog.favoriteToy = "tug-of-war rope";

So, like before, we must create helper methods in the Dog class to change this information. We call these methods "setters".

Dog.java

...
public void setAge(int age) {
this.age = age;
}
public void setFavoriteToy(String favoriteToy) {
this.favoriteToy = favoriteToy;
}

Notice how we did not create a setter for "name". Names are unlikely to change, so we typically keep that kind of information for our records only. Now let's replace that ruined toy!

Application.java

public class Application {
public static void main(String args[]) {
// Store the dog's name in a variable for reusability.
String name = dog.getName();
...
// Update the dog's information using setters.
dog.setFavoriteToy("tug-of-war rope");
// Print the dog's information using getters
System.out.println(name + "'s new favorite toy is a " + dog.getFavoriteToy() + "!");
}
}

Output

...
Molly's new favorite toy is a tug-of-war rope!

Now we can see that the toy is replaced and Molly is happy again!
